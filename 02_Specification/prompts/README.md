# Prompts для Claude Code

Эта папка содержит **готовые промпты** для копирования в Claude Code. Каждый файл — самодостаточный промпт для одного действия (Phase B setup или один sprint реализации).

## Как пользоваться

**Шаг 1.** Откройте папку проекта в терминале (Windows Terminal или PowerShell):
```bash
cd C:\Users\Николай\Desktop\Проект B787
```

**Шаг 2.** Запустите Claude Code:
```bash
claude
```

**Шаг 3.** Откройте нужный промпт-файл из этой папки в любом редакторе (Notepad, VS Code, или прямо здесь в чате).

**Шаг 4.** Скопируйте **всё содержимое** промпт-файла (Ctrl+A → Ctrl+C).

**Шаг 5.** Вставьте в окно Claude Code (Ctrl+V) и нажмите Enter.

**Шаг 6.** Дальше следуйте инструкциям, которые Claude Code будет давать (он будет читать спеку, делать summary, ждать вашего «go», писать код).

## Workflow одного спринта (для понимания)

После того как вы вставили один из sprint-промптов в Claude Code, происходит следующее:

```
[1] Claude Code читает спеку
    └→ выводит summary в 5–10 пунктов
    
[2] Вы пишете "go"
    └→ Claude начинает реализацию
    
[3] Claude пишет код, тесты, коммитит, пушит
    └→ открывает PR через `gh pr create`
    
[4] GitHub Actions запускает CI на PR
    └→ зелёные галочки = всё ОК
    
[5] Вы открываете PR в браузере
    └→ читаете "Manual testing instructions"
    
[6] Вы открываете промпт `99-pr-review-template.md`
    └→ копируете и следуете инструкциям для проверки в Expo Go
    
[7] Если работает — напишите `merge it` в чат Claude Code
    └→ Claude Code сам выполнит merge и удалит ветку
    
[8] Переходите к следующему sprint-промпту
```

## Когда что-то идёт не так

**Claude Code говорит «не могу понять X».**
Это работает по правилам из `AGENTS.md` Rule 2 — стоп при неоднозначности. Прочитайте, что он не понял, посмотрите спеку, ответьте уточнением. Не торопите его «делай как угодно» — пусть лучше уточнит.

**CI красный (одна из проверок не прошла).**
Откройте PR, посмотрите какая проверка красная, кликните на неё, прочитайте логи. Скорее всего — typo, missing test, или lint-нарушение. Скопируйте сообщение об ошибке, вставьте в Claude Code: «Видишь это в CI? Почини». Он исправит.

**Expo Go показывает экран ошибки или не загружается.**
Это обычно проблема в коде (синтаксическая, runtime). Скопируйте текст ошибки из Expo Go, вставьте в Claude Code. Он диагностирует и починит.

**Что-то пошло сильно не по плану.**
Вы можете сказать Claude Code: «Откати последний commit и попробуй другой подход». Если PR ещё не замержен — это безопасно. Если замержен — уже сложнее, но `git revert` тоже работает.

## Важные файлы за пределами этой папки

Каждый промпт ссылается на конкретные документы спеки. Эти ссылки — обязательное чтение для Claude:

- `02_Specification/AGENTS.md` — правила работы агента (читается всегда).
- `02_Specification/01-vision.md` … `02_Specification/10-glossary.md` — спека.
- `02_Specification/ADR/*.md` — обоснование решений.
- `02_Specification/module-contracts/*.md` — публичные API модулей.

Если в каком-то промпте указано «прочитай файлы X, Y, Z» — это критично, не пропускайте этот шаг.
