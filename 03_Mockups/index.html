6. Технический стек и платформа

Стек выбран с приоритетами: (а) разработка ведётся на Windows 11, (б) Mac не требуется на любом этапе работы, (в) гарантированное прохождение модерации App Store, (г) долгосрочная поддержка и расширяемость, (д) Clean Architecture сохраняется концептуально.

Платформа

Apple iPad · iPadOS 16.0+
Поддержка ~95% активных iPad
iPad-first дизайн (адаптируется на iPhone опционально)

Язык и фреймворк

TypeScript · строгая типизация
React Native + Expo SDK · UI и навигация
React Navigation для роутинга экранов
Reanimated для анимаций

Архитектура

Clean Architecture + MVVM
Слои: Presentation / Domain / Data
Модули через структуру папок: src/features/<module>/{domain,data,presentation}
Правило: features не зависят друг от друга, только от core

Сборка и распространение

EAS Build · сборка iOS .ipa в облаке Expo
EAS Submit · автосабмит в App Store Connect
Public App Store · бесплатно · Individual account ($99/год)
Mac не требуется ни на одном этапе

Среда разработки

Windows 11 · основная рабочая ОС
VS Code или Cursor + Claude Code
Node.js 20 LTS, npm/pnpm
Тестирование на iPhone через Expo Go

Хранение данных

Полностью локально · 100% офлайн
Bundled JSON (опорные значения, импорт через TypeScript)
AsyncStorage / expo-secure-store (настройки)
Без серверного backend, без сетевых запросов

Локализация

EN + RU на старте
i18next + react-i18next (стандарт RN-экосистемы)
Авиационные термины (KT, MAC, RWYCC) — без перевода

Тестирование

Jest · unit-тесты Domain-слоя
React Native Testing Library · UI-тесты
30+ acceptance-кейсов на алгоритм Crosswind
Beta с пилотами через TestFlight (Expo поддерживает)

Privacy / Compliance

Zero data collection
Privacy Manifest (PrivacyInfo.xcprivacy) — Expo генерирует
Privacy Policy + Terms of Use на отдельной странице
GDPR-friendly by design
Почему Mac не нужен. EAS Build — облачный сервис Expo, который выполняет финальную сборку iOS-приложения на Apple Silicon Mac-серверах в датацентре Expo. Вы со своего Windows 11 запускаете команду eas build --platform ios, через 10–15 минут получаете подписанный .ipa-файл, и команда eas submit отправляет его в App Store Connect. Бесплатный тариф EAS обычно покрывает потребности проекта; платный план $19/мес — если будет много билдов.

7. План реализации (последовательность этапов)

Поэтапная структура работ от подготовки до публикации в App Store. Этапы идут последовательно; разработка ускоряется за счёт AI-ассистента (Claude Code) и простоты MVP-функционала. Конкретные сроки уточняются в ходе работы.

Phase 0 · Подготовка
Регистрация Apple Developer Program (Individual, $99/год) · сбор опорных значений и финальной матрицы операционных лимитов · оформление письменного подтверждения от руководства авиакомпании (рекомендуется) · финализация дизайн-системы и App Store Listing-черновика.
Phase 1 · MVP разработка
Скаффолд проекта (App + Core + Crosswind feature module + Design System) · реализация Crosswind Landing с unit-тестами · Splash + Disclaimer + Main Menu + Settings + About · локализация RU+EN · Privacy Manifest · acceptance-тестирование с действующими B787-пилотами через TestFlight.
Phase 2 · App Store submission
Подготовка App Store Connect (метаданные, скриншоты, App Preview видео, Review Notes со ссылками на аналоги) · TestFlight внутренняя бета (минимум 5 пилотов) · Submit → Review → Release.
Phase 3 · Crosswind Takeoff
Добавление модуля для взлёта. Общая логика расчёта переиспользуется, отдельные таблицы операционных значений.
Phase 4+ · Расширение
Weight & Balance · Performance V-speeds · Fuel Planning · опционально другие воздушные суда (B737, B777). Каждый модуль — отдельный SPM-пакет, не затрагивающий существующий код.

8. Ключевые решения и готовые рекомендации

Принципиальные моменты проекта. Большинство уже имеет рекомендуемое решение — отметка «принято» означает, что на текущем этапе обсуждения позиция зафиксирована и может быть пересмотрена только при веских причинах.

D-01
Распространение: Public App Store
Принято. Бесплатное публичное приложение, доступное любому B787-пилоту глобально. Custom App / Apple Business Manager не используется (авиакомпания не оформляет аккаунт ABM).
принято
D-02
Apple Developer Account: Individual
Принято. Регистрация на физическое лицо разработчика, юрисдикция Узбекистан, $99/год. Авиакомпания не вовлечена в Apple Developer Program — приложение является личным проектом разработчика.
принято
D-03
Позиционирование данных: «conservative advisory limits»
Рекомендация (для одобрения App Store): в публичных текстах приложение позиционируется как «Conservative advisory limits for Boeing 787, validated with active line pilots. Limits are intentionally more restrictive than Boeing demonstrated values for additional safety margin.» Конкретная авиакомпания НЕ упоминается в App Store Listing, About, дисклеймере. Это устраняет любые вопросы App Reviewer-а о permission на использование чужих данных.
рекомендация
D-04
Письменное подтверждение от руководства
Рекомендация для долгосрочной защиты разработчика. Достаточно email или короткой записки от Flight Operations / руководства авиакомпании, подтверждающей разрешение на использование операционных значений в личном проекте мобильного приложения. Не публикуется, хранится у разработчика. Не требуется для App Store, требуется для собственной защиты.
рекомендация
D-05
Брендинг — нейтральный авиационный
Принято. Без логотипов, цветов или явных отсылок к конкретной авиакомпании. Тёмно-зелёный/teal акцент, SF Pro / SF Mono шрифты, минималистичная HIG-эстетика. Это и юридически чисто, и универсально для любого B787-пилота.
принято
D-06
Финальный текст дисклеймера
Версия для splash-экрана: «Advisory only. Calculations provide conservative reference values for Boeing 787 operations. Final operational decisions must always be based on official Boeing FCOM/QRH and your operator's procedures. Not for primary navigation or operational use.» Тестируется при первом запуске и при major-обновлениях.
черновик
D-07
Бета-тестирование с пилотами
5–10 действующих пилотов B787, готовых тестировать через TestFlight перед публичным релизом. Цель — валидация UX, корректности расчётов и удобства в реальных рабочих сценариях (брифинг-рум, кокпит).
Phase 1
D-08
Roadmap следующих модулей
Приоритизация Phase 3+ определяется по обратной связи после релиза MVP: Crosswind Takeoff (расширение текущего модуля) vs Weight & Balance vs Performance V-speeds vs Fuel Planning. Архитектура позволяет добавлять модули независимо.

9. Подтверждённые аналоги в Public App Store

Эти приложения уже опубликованы в Public App Store и работают по тому же принципу, что предлагаемый B787 Calculator: advisory-калькуляторы для коммерческой авиации с использованием данных производителя самолёта. Их успешное существование доказывает реализуемость нашего подхода.

Boeing OPT

FROM BOEING
id 584211407 · Boeing
Официальный Onboard Performance Tool от Boeing. Реал-тайм takeoff/landing analysis, weight & balance. Поддерживает B787-9 takeoff limitations с версии 4.2. Прямое доказательство, что Apple одобряет performance-калькуляторы для B787 в публичном App Store.

B737 Performance Handbook

3RD PARTY
id 871314271 · independent dev
Это самый близкий аналог нашего проекта. Сторонний разработчик; landing distance, brake cooling, go-around calculator для всех B737 моделей (CL/NG/MAX). Годами в App Store. Доказательство, что независимый dev может опубликовать аviation calculator для конкретной модели Boeing.

B737 Performance

3RD PARTY
id 6752240605 · Burak Kara
Performance-калькулятор для Boeing 737-800W CFM56-7B26. Точные takeoff/landing data. Прямой пример соло-разработчика, опубликовавшего performance calculator конкретной модели Boeing в Public App Store.

iBOEING B-787

3RD PARTY
id 1629240798 · independent dev
Type rating prep и systems training именно для Boeing 787 Dreamliner. Включает aviation calculator с конвертерами и полётным планированием. Существующий B787-специфичный продукт в App Store.

E6B Aviation Calculator

CLASSIC
id 286046305 · Aviation Mobile Apps
Универсальный авиационный калькулятор. 75+ aviation calculations, unit conversions, weather. Высокий рейтинг, многолетнее присутствие в Public App Store. Образец того, как должен быть оформлен advisory aviation tool.

ForeFlight Mobile EFB

EFB LEADER
id 333252638 · ForeFlight LLC
Один из лидеров EFB-приложений на App Store. Распространяется публично. Подтверждение, что серьёзные операционные инструменты для пилотов работают в Public App Store при правильном positioning и compliance.
Ключевой вывод для руководства: Public App Store — стандартный путь для авиационных advisory-калькуляторов, по которому уже идут как сам Boeing (с OPT), так и независимые разработчики. Наш проект следует проверенному паттерну, не изобретает регуляторного велосипеда и опирается на годы успешной практики на платформе. Apple одобряет такие приложения при соблюдении трёх условий: явный advisory-дисклеймер, точное описание в App Store Listing, ссылка на FCOM/QRH как primary source.

10. App Store compliance — стратегия прохождения

Конкретные действия для гарантированного прохождения App Store Review. Основано на анализе аналогичных приложений (B737 Performance Handbook, Boeing OPT, Maverick), которые уже опубликованы в App Store.

Disclaimer Strategy

  • Splash-экран с обязательным подтверждением
  • «Advisory» chip на каждом результате
  • Reference: FCOM 4.20 видим всегда
  • Дисклеймер при major-обновлениях

Privacy by design

  • Zero data collection
  • PrivacyInfo.xcprivacy в bundle
  • Без сторонних SDK / аналитики
